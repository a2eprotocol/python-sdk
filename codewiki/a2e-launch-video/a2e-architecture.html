A2E Protocol Architecture

Agent-to-Environment Protocol — Three-layer design with 10 capability namespaces

Why A2E?

  • • Standard protocol for agent-environment interaction
  • • 10 capabilities, one wire format
  • • Stateful sessions with snapshot/restore
  • • Plugin-based — thin kernel, dynamic loading

Key Integrations

  • • MCP Bridge — use any MCP server as an A2E tool
  • • RL environments — Gymnasium-compatible step loops
  • • HTTP+SSE + DirectTransport
  • • Pydantic v2 schemas
